L'ordre d'execution du code par le navigateur

Par défaut, un navigateur va lire et exécuter le code dans l'ordre de son écriture.

Lorsque le navigateur arrive à un élément script, il va stopper le traitement du reste HTML jusqu'a ce que le code JavaScript soit chargé dans la page et exécuté.

résumons ce qu'il se passe dans ce cas.

  1. Le navigateur commence à analyser (ou à traiter) le code HTML.
  2. Le navigateur rencontre un élément script
  3. Le contenu JS est demandé et téléchargé (si il est en externe) puis executé. Durant tout ce temps, l'analyseur du navigateur bloque l'affichage du HTML ce qui peut ralentir si le sript est long, l'affichage de la page.
  4. Dès que le JS à été executé, le contenu HTML est affiché.

Ce souci d'attente est résolu grâce au téléchargement "asynchrone" des données avec l'attribut asyncou defer dans le cas où l'ont fait appel à des fichiers externes.

Le point virgule

le point virgule ; est utilisé pour signifié la fin d'une instruction ( une variable, une condition, une boucle etc...).
